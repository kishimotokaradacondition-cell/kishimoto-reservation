# コンサルティング事前決済（Stripe）セットアップ手順

きしもとマネジメントオフィスのコンサルティング予約（`/consulting`）で、
クレジットカードによる事前決済を有効にするための手順です。

## 仕組み

- お客様が `/consulting` で枠を選び情報を入力すると **仮予約** が作成され、
  Stripe の決済ページに移動します
- **支払い完了で予約が確定**し、確認メール・各種通知が送信されます
- 30分以内に支払いが完了しない場合、仮予約は自動で解放されます
- Stripe 未設定の間は、`/consulting` も通常予約（決済なし）として動作します

## 1. Stripe アカウントの作成

1. https://stripe.com/jp でアカウントを作成
2. 事業情報（屋号・住所・銀行口座・本人確認）を登録して審査を受ける
   （審査完了までは「テストモード」で動作確認できます）

## 2. API キーの設定

Stripe ダッシュボード →「開発者」→「API キー」から
**シークレットキー**（`sk_test_...` または `sk_live_...`）を取得し、
デプロイ先（Heroku 等）の環境変数に設定します。

| 環境変数 | 内容 | 必須 |
|---|---|---|
| `STRIPE_SECRET_KEY` | Stripe シークレットキー | ✅（これを設定すると事前決済が有効化） |
| `STRIPE_WEBHOOK_SECRET` | Webhook 署名シークレット（`whsec_...`） | 推奨 |
| `CONSULTING_PRICE` | カウンセリング料金（円・税込、既定値 10000） | 任意 |
| `BASE_URL` | 公開URL（例 `https://example.com`）。未設定時は自動判定 | 任意 |

`config.py` を使っている場合は、同名の変数を `config.py` に書いても動作します。

## 3. Webhook の設定（推奨）

決済ページを閉じてしまった場合でも確実に予約を確定させるため、
Stripe ダッシュボード →「開発者」→「Webhook」でエンドポイントを追加します。

- URL: `https://＜あなたのドメイン＞/api/stripe/webhook`
- 受信イベント: `checkout.session.completed`, `checkout.session.expired`
- 発行された署名シークレット（`whsec_...`）を `STRIPE_WEBHOOK_SECRET` に設定

※ Webhook 未設定でも、決済完了後の戻りページ（`/payment/success`）で
予約確定される二重の仕組みになっていますが、Webhook 設定を推奨します。

## 4. 動作確認（テストモード）

1. `STRIPE_SECRET_KEY` にテストキー（`sk_test_...`）を設定
2. `/consulting` から予約し、テストカード番号 `4242 4242 4242 4242`
   （有効期限は未来の任意日付、CVC は任意の3桁）で支払う
3. 予約確定ページ・確認メール・管理画面の「支払い」欄（決済済）を確認
4. 問題なければ本番キー（`sk_live_...`）に差し替え

## 返金について

キャンセル時の返金は Stripe ダッシュボード →「支払い」から
該当の決済を選んで「返金」で行えます（手動運用）。
